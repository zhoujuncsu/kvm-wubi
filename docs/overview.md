## 概览

这个项目解决的问题：把“要输入的文本”可靠地输入到 KVM/远程窗口中，同时在中文五笔输入法、英文/网址/邮箱、中文标点之间自动切换，尽量减少重码选词造成的误输入。

### 分层

- 转换层（Translation）
  - 入口：translate.py 的 WubiTranslator
  - 策略：单字模式（逐字查 wubi.json），避免词组重码带来的候选不确定性
  - 输出：token 列表（每个 token 含 text、code、type）
- 执行层（Execution）
  - 入口：main_gui.py（GUI 内部执行输入）
  - 策略：根据 token.type 决定是否切换中/英模式、按键组合、提交方式（空格/数字选词）

### 输入法模式约定

- 中文模式：目标主机处于五笔输入法中文状态
  - token.type 为 wubi/chinese_punct/chinese_punct_shift 时使用
- 英文模式：目标主机处于英文直输状态
  - token.type 为 raw（ASCII）时使用

模式切换方式：通过模拟按一次 Shift 键切换（与目标主机输入法配置相关）。

### 单字模式的重要性

对于许多五笔输入法，实现会将“单字”和“词组”一起参与候选排序。即使编码正确，也可能因为候选顺序不同导致选到词组（例如“高”与“调味”同码）。目标主机开启“单字模式”可以从输入法侧强制只给出单字候选，从根源减少重码问题。

### 纠错表 corrections.json

即便在单字模式下，仍可能存在单字与单字重码或候选顺序不符合预期的情况。此时可通过外置 corrections.json 指定：

- `"字": "编码"`：直接覆盖该字编码
- `"字": "编码N"`：附带候选序号（例如 qqu2），用于输入法候选选择
